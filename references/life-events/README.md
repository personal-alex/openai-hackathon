# Life-event references

Store curated, event-level reference material here. This material describes the human journey before it describes an institution.

## Suggested layout

```text
life-events/
  first-child/
    overview.md
    questions.md
    roadmap-template.json
    sources.md
  job-loss/
  marriage/
```

## Each event package should include

- Event definition and boundaries
- Candidate user-context fields
- High-value clarifying questions
- Roadmap-step candidates and dependencies
- Jurisdiction-neutral assumptions, clearly marked
- Links to jurisdiction-specific sources kept in `../regulations/`
- Evaluation cases and known edge cases

Do not treat examples or templates as legal, medical, tax, or financial advice. Record uncertainty explicitly.
