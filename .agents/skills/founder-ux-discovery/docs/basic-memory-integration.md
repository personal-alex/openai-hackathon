# Project Memory Integration

Use this protocol with `basic-memory` or another configured memory system.

## Before discovery

Read only relevant context: vision, users, workflows, decisions, rejected directions, assumptions, open questions, and constraints. Follow the project’s actual instructions, tool names, and memory project. Never invent API calls or claim retrieval that did not happen.

## During discovery

Label retrieved context separately from new facts, founder assumptions, UX hypotheses, and open questions. Surface conflicts instead of silently selecting a version.

## After meaningful work

Proactively save durable knowledge: summaries, journeys, models, concepts considered, rejected options, selected direction, decisions, trade-offs, and open questions. Avoid storing trivial chat detail.

## Resume

Read relevant memory first, summarize known facts, decisions, and unknowns, then continue from the highest-value unresolved question rather than restarting.

## Decision format

```text
UX Decision:
- Decision: [decision]
- Context: [context]
- Alternatives considered: [alternatives]
- Reason: [reason]
- Trade-offs: [trade-offs]
- Consequences: [consequences]
```
