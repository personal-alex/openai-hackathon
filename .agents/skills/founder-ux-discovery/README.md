# Founder UX Discovery

A Codex skill for taking a founder from a product idea to an evidence-based UX direction before visual design or implementation.

## Workflow

1. Read relevant configured project memory.
2. Interview the founder one question at a time.
3. Validate the problem and user model.
4. Explore 3–5 distinct interaction models.
5. Compare trade-offs and recommend one direction.
6. Create a UX specification and document durable decisions.
7. Hand off to information architecture, Impeccable, and implementation as appropriate.

Suggested prompts: “Help me discover the UX for this product idea”, “Explore different UX models”, “Compare the concepts”, “Recommend a direction”, “Create the UX specification”, or “Continue our prior UX discovery.”
