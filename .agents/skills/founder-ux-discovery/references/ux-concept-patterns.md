# UX Concept Patterns

Use these to produce genuinely different interaction models.

| Pattern | Best for | Principal risk |
|---|---|---|
| Timeline | events, milestones, deadlines | implies a linear reality |
| Guided journey | complex, unfamiliar, high-load processes | restricts experts or feels slow |
| Conversational assistant | ambiguous situations and personalization | hidden capabilities, weak discoverability |
| Case manager | multi-step situations, documents, and tasks | feels bureaucratic |
| Task manager | execution and clear next actions | hides rationale |
| Workspace | professional, multi-object workflows | high initial complexity |
| Search-first | known-item access in large information spaces | fails users who do not know what to seek |
| Map or graph | dependencies and non-linear exploration | visual overload and poor mobile fit |
| Hybrid | distinct problems need distinct models | becomes indecisive complexity |

For every concept, describe its metaphor, user mindset, primary interaction, typical flow, conceptual structure, strengths, weaknesses, best context, and main risk.
