# Founder UX Discovery — return orientation

## Founder assumption

On returning after a few days, the most prominent information should be **what changed since the last check and/or progress**, rather than only the next action or a generic overview.

## UX implication (hypothesis)

The return state should briefly re-orient the person with a meaningful delta before landing them on the current attention horizon. This needs clarification because change may mean user-marked progress or a route update after context changes; the MVP has no external integrations or background updates.

## Open question

Does “what changed” primarily mean progress the person marked themselves, or route changes after they update their situation?
