# Founder UX Discovery — explorable route

## Confirmed founder preference

When viewing a route, people should primarily be free to **explore the whole route and choose where to focus**, rather than being led through a single prescribed current action.

## UX implication (hypothesis)

The product should show a clear current attention horizon without locking navigation or forcing a linear wizard. A coherent route overview, task inspection, and context-update path matter more than step-by-step completion choreography.

## Trade-off

Freedom needs a strong default orientation so users do not return to the original “too many things to do” problem.
