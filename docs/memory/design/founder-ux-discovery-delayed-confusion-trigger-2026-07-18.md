# Founder UX Discovery — delayed-confusion trigger

## Founder assumption

Most people will first open Life Navigator **after a few days of confusion**. Some will arrive immediately after the event; relatively few will first arrive only when a concrete request or paperwork item appears.

## UX implication (hypothesis)

The first experience should feel like rapid relief and orientation rather than a form-heavy intake. It should acknowledge that the user may already have searched, delayed, or partially acted, then quickly establish a current route without requiring a complete retrospective.

## Open question

At this point, is the stronger fear that there are too many possible actions, or that an important action has already been missed?
