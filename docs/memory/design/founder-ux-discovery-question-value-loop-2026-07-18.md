# Founder UX Discovery — question value loop

## Confirmed founder insight

A Life Navigator question is worth answering when the person can see:

1. **Why it is being asked** before answering; and
2. **Its effect on the roadmap** after answering.

## UX implication (hypothesis)

Questioning should be a visible refinement loop, not a hidden intake process: question rationale → answer/unknown → validated route update with an understandable change. The UI must not invent the reason or change explanation; both come from approved/validated data.

## Open question

Must a useful initial route be visible before any refinement question is answered?
