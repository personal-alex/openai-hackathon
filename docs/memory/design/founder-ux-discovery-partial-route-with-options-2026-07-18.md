# Founder UX Discovery — partial route with options

## Confirmed founder decision

When the product lacks enough information for a complete route, it should **show whatever validated portion it has** and explain the remaining options, rather than withholding the route or offering a generic checklist.

## UX implication (hypothesis)

The route should represent known actions, unknown-dependent branches, and the one question that would resolve a branch. This must not make unselected paths look like obligations or predictions; each remains visibly conditional until an approved answer validates it.

## Emerging UX principle

**Show the map honestly, including its unresolved territory, so the person can regain control without being misled.**
