# Founder UX Discovery — trust model

## Confirmed founder insight

Trust in a Life Navigator action rests on three jointly important elements:

1. **Official sources** — visible provenance is a major reason to trust the route.
2. **Why it applies** — the user needs to understand the connection between their situation and an action.
3. **Ability to tweak assumptions** — the user must be able to correct the route's context rather than accept it as opaque authority.

## UX implication (hypothesis)

Every action needs lightweight provenance and relevance signals, with deeper source/review evidence and context editing available without making the initial route dense. The UI must make clear that an action is planning support and verification, not an official determination.

## Open question

Should provenance be visible directly on every route step at first glance, or be readily inspectable when the user asks for it?
