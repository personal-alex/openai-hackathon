# Founder UX Discovery — attention horizon

## Confirmed founder preference

The whole route should first communicate **what needs attention now versus later**.

## UX implication (hypothesis)

The main route should use a small number of attention horizons, with immediate actions visually distinct from preparation and later steps. The route must remain a planning horizon rather than a personal deadline engine.

## Safety constraint

Labels, timing, and any verification framing must render only validated catalog/compiler data. The UI must not infer urgency, policy deadlines, or eligibility.
