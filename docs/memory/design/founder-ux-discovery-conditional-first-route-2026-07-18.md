# Founder UX Discovery — conditional first route

## Confirmed founder decision

Show a meaningful initial route before refinement questions **when such a validated version exists**. Do not assume one will always be available.

## UX implication (hypothesis)

The entry experience needs two honest paths:

- Provisional route available → reveal it early and explain that questions refine it.
- Insufficient validated context → identify the one most decision-changing missing input needed to begin; do not fabricate a generic route or hide uncertainty behind an empty state.

## Open question

When no meaningful route can be shown yet, should the product explain the one fact it needs to get started, or provide a broad generic checklist as a temporary fallback?
